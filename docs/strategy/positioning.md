# AgentGuard Positioning

## One-line positioning
AgentGuard is an action firewall for AI agents that enforces policy before any tool or MCP action executes.

## Who this is for
- Enterprise security teams responsible for AI risk and controls.
- Platform engineering teams operating internal agent infrastructure.
- Developer productivity teams deploying autonomous coding/research agents.

## Why they buy
- Prevent unsafe agent actions before they run.
- Reduce prompt/tool/MCP injection blast radius.
- Provide audit-grade decision evidence for security and compliance.
- Control autonomy with enforceable policy instead of trust-only workflows.

## Differentiation vs guardrails-only products
- Action-first enforcement: decisioning happens at tool execution boundaries, not just prompt text.
- Unified policy engine: prompt injection, SQLi, code injection, exfiltration, rate, and scope in one decision.
- Security operations fit: audit logs and deterministic deny reasons suitable for SOC workflows.

## Core value proposition
Move fast with autonomous agents while enforcing least privilege, runtime controls, and investigation-ready telemetry.
