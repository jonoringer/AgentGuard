# AgentGuard Architecture v2

## Objectives
- Enforce security decisions at agent action boundaries.
- Support deterministic controls and adaptive risk scoring.
- Produce durable, queryable audit evidence.

## Runtime flow
1. Agent submits action intent.
2. Policy engine runs detector pipeline.
3. Detector findings are scored into risk + confidence.
4. Action mapper returns `allow`, `review`, `quarantine`, or `deny`.
5. Decision and evidence are persisted and streamed.

## Core components
- API Gateway: receives `/v1/guard/evaluate` actions.
- Policy Engine: orchestrates checks and scoring.
- Detector Pipeline: pluggable detectors (prompt, SQLi, code, exfiltration, scope, rate).
- Decision Mapper: converts risk score to enforcement action.
- Audit Service: persists and streams decisions.
- Export Service: sends events to SIEM/telemetry sinks.

## Decision contract
- `decision`: binary compatibility field (`allow`/`deny`).
- `enforcement_action`: `allow`/`review`/`quarantine`/`deny`.
- `risk_score`: integer 0-100.
- `confidence`: float 0-1.
- `rule_results`: list of rule outcomes with severity.

## Data and observability plan
- Short term: in-memory + SSE for rapid iteration.
- Next: PostgreSQL audit store with indexed query keys.
- OTel traces and metrics around evaluation latency and detector hit rates.

## Security governance roadmap
- Policy versioning and approvals.
- Multi-tenant isolation and RBAC.
- Exception workflows and time-bound policy overrides.
